## STEP 0：設計ドキュメントを GitHub に上げる（最初の作業）

Claude Code に下のプロンプトをそのまま貼ってください。
ローカルのファイルを読み込み、リポジトリに push（＝GitHubへアップロード）するところまで案内・代行してくれます。

```
これから ARSENAL FRONT というゲームを Godot 4 で作ります。
私はコードを一切書けず、ターミナル操作にも不慣れです。すべてあなたに任せます。

まず、私のPCの次のフォルダに設計ドキュメントが5つ入っています：
C:\Users\tkmuh\Downloads\ARSENAL-FRONT-docs-with-prompts
（PLAN.md / SPEC.md / TODO.md / KNOWLEDGE.md / claude_code_prompts.md）

これらを GitHub リポジトリ https://github.com/Sierra-117605/ARSENAL-FRONT に
アップロード（push）したいです。リポジトリは現在空です。

以下を順番にやってください：
1. 作業フォルダの方針を決める（上記フォルダをそのまま使うか、別の場所にリポジトリをcloneして
   そこへファイルをコピーするか）。どちらが良いか理由とともに提案してください。
2. 必要なら git の初期設定やリポジトリの clone を、コマンドを1つずつ
   「何のためのコマンドか」を日本語で説明しながら案内してください。
3. 5つのファイルをリポジトリに追加し、commit して push してください。
4. push に GitHub の認証が必要な場合、私のPCに認証が設定済みかどうかを確認する方法と、
   未設定なら設定手順を日本語で案内してください（パスワードやトークンは私自身が入力します）。

各ステップの前に「今から何をするか」、各ステップの後に「何が起きたか」を日本語で説明してください。
```

> 補足：push の最後で GitHub 認証を求められることがあります。そのトークンやパスワードは
> あなた自身が入力します（Claude Code は代わりに入力しません）。未設定でも手順を案内してくれます。

## STEP 1：プロジェクト全体の把握

アップロードが終わったら、これを貼ります。

```
リポジトリのファイルがそろいました。
PLAN.md / SPEC.md / TODO.md / KNOWLEDGE.md の4つの設計ドキュメントをすべて読み込んで、
プロジェクトの全体像を把握してください。
（claude_code_prompts.md は私への手順書なので、参考程度でOKです）

重要な前提：
- 私はコードを読めない・書けません。検証は「実際に動かして結果を見る」方法でしか行えません。
- 実装は Claude Code（あなた）と Codex の2つで行い、相互レビューで品質を担保する方針です。
- 描画は2D固定、エンジンは Godot 4 + GDScript、データ構造はマルチプレイ将来対応のためシリアライズ可能に。

読み終えたら、以下を日本語で報告してください：
1. このプロジェクトが何を作ろうとしているかの要約
2. Phase 1 のゴール（TODO.md記載）の確認
3. 最初に着手すべきタスク（TODO.md の現在地）
まだコードは書かず、報告と着手プランの提示までにとどめてください。
```

## STEP 2：実装開始（T1-1 から）

STEP 1 の報告に納得したら、これを貼って実装を始めてもらいます。

```
報告内容でOKです。では TODO.md の「現在地」に従って T1-1（Godotプロジェクト作成）から実装を始めてください。

進め方のルール：
- 1タスクずつ進め、完了したら TODO.md の該当行のチェック記号を更新してください。
- 各タスクの終わりに「何を作ったか」「私が動作確認するには何をどう操作すればいいか」を日本語で説明してください。
- 詰まったり判断に迷ったら、勝手に仕様を決めず、KNOWLEDGE.md に記録した上で私に質問してください。
- SPEC.md に [TBD]（未確定）と書かれた数値などにぶつかったら、仮の値で進めて構いませんが、何を仮置きしたか必ず報告してください。
- Godot エディタ側で私が手作業する必要がある場面は、クリック手順を日本語で1つずつ説明してください。
```

## STEP 3：区切りごとに使う定型プロンプト

### 動作確認のしかたを聞く
```
今できたところまでを、私が自分のPCで実際に動かして確認したいです。
何をどの順でクリック・操作すればいいか、日本語で手順を教えてください。
```

### Codex にレビューさせたいとき（相互チェック運用）
```
今書いたコードを、私が Codex にレビューさせたいです。
Codex に渡すための「レビュー依頼文」と「対象ファイルの場所」を日本語でまとめてください。
レビューで見てほしい観点（バグ・設計・将来のマルチプレイ対応など）も箇条書きで添えてください。
```

### 作業を中断して後で再開するとき
```
今日はここまでにします。次回このプロジェクトを再開したときに、
あなた（または別のセッション）が状況をすぐ把握できるよう、
TODO.md の「現在地」セクションを最新状態に更新してください。
今どこまで終わって、次に何をやるべきかが分かるようにしてください。
```

### 成果を GitHub に保存（push）したいとき
```
ここまでの成果を GitHub リポジトリに保存（push）したいです。
私はコマンドに不慣れなので、必要な操作を1つずつ、
何のためのコマンドかを説明しながら案内してください。
push に認証が必要なら、その手順も日本語で教えてください。
```

## 困ったときのコツ
- うまくいかないときは「いま何が起きているか」を画面のまま日本語で伝えるだけでOK。エラーメッセージはコピペで渡す。
- 「コードの意味は説明しなくていいので、私がやる操作だけ教えて」と言えば、説明を操作手順に絞ってくれます。
- 仕様を変えたくなったら、Claude Code ではなく claude.ai（仕様策定チャット）に戻って4ドキュメントを更新 →
  GitHub に再アップロード → Claude Code に「ドキュメントが更新されたので読み直して」と伝える、の順が安全です。
